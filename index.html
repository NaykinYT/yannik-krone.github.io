Computational Neuroscience · Representation Learning

Mapping neuronal activity to behavior—ML models that predict how interneurons in C. elegans encode locomotion and chemotaxis from calcium imaging and behavioral recordings.

I hold a paid Research Assistant position—a full researcher role—at the Laboratory of Physics of Biological Systems (LPBS) under Prof. Sahand Jamal Rahi. I develop machine learning models to predict behavioral responses directly from neuronal activity. My focus is on the predictive modeling side: mapping time-series of neuronal activity to future and past behavior.

Working with C. elegans as a model organism, we use simultaneous neuronal and behavioral recordings from freely moving worms during chemotaxis. Second-layer interneurons (RIA, RIB, RIM) integrate sensory and internal signals; the goal is to understand how they encode behaviorally relevant information. This sits at the intersection of applied machine learning, computational neuroscience, and time-series modeling—combining scalable learning systems with interpretable representations of how neural activity maps to behavior over time.

Separator-Aware SAT Encoding for Exact Treewidth

Algorithms · Graph Theory · SAT/MaxSAT

A SAT-based exact treewidth solver that exploits safe graph separators—articulation points, cliques, almost-cliques—to decompose instances and encode width constraints, achieving empirical speedups over baseline encodings.

Code Paper
Click for details

Context: 12 ECTS research project (nearly half a semester) at the INDY Lab (Information and Network Dynamics) at EPFL, supervised by Prof. Matthias Grossglauser and Prof. Patrick Thiran. Grade: 6/6.

Treewidth is a central graph parameter measuring how "tree-like" a graph is; computing it exactly is NP-hard, yet bounded treewidth enables efficient algorithms for many NP-hard problems (CSP, SAT, probabilistic inference, etc.). This project implements a SAT-based exact treewidth solver that embeds safe graph decompositions into an incremental SAT/MaxSAT framework.

The key innovation is a separator-aware encoding that exploits graph structure before encoding:

  • Safe separators: Articulation points, clique separators, and almost-cliques are used to decompose the graph into smaller subinstances. Only bags that respect these separators are encoded, reducing the search space while preserving correctness.
  • Globally linked width constraints: Width constraints are integrated across subinstances so that the encoding remains correct when solving decomposed fragments. This avoids invalid decompositions that can arise in naive decomposition-based encodings.
  • Incremental framework: The solver reuses learned clauses across iterations (binary or linear search over the target width), and supports both SAT (decision) and MaxSAT (optimization) formulations.

The implementation (Python 3.13+) supports multiple strategies: iterative binary search, iterative linear search, and MaxSAT—in both standard Berg–Järvisalo style and separator-aware variants. A hybrid solver combines separator decomposition with heuristic lower/upper bounds and SAT solver enhancements. Benchmarked on PACE 2017 instances, Berg–Järvisalo graphs, and random graphs, the approach achieves consistent empirical speedups over baseline SAT encodings, demonstrating that structure-aware decomposition pays off for exact treewidth computation.

Fine-Tuning Compact LLMs for STEM Education

NLP · Large Language Models · Efficient Training

Can a small LLM tutor STEM offline? Fine-tuned Qwen3-0.6B for efficient deployment using DPO, LoRA, quantization, and RAG—exploring how far sub-billion models can go for specialized tutoring.

Paper Code not publicly available (academic reasons)
Click for details

Course: CS-552 Modern Natural Language Processing — taught by Prof. Antoine Bosselut. Group project of 4.

The project asks: can a small LLM serve as a reliable STEM tutor in resource-constrained settings (e.g. offline, battery-limited)? We fine-tuned Qwen3-0.6B-Base using four interconnected components, targeting efficient deployment for educational use. A key finding: data quality outperformed quantity, and small models benefit from pipelines that diverge from standard large-model recipes.

  • Reward model: Aligned the base model with human preferences using Direct Preference Optimization (DPO). Trained on mixed datasets (M1 human labels, HH-RLHF, Nectar, SHP, UltraFeedback, Math Stack Exchange). Best setup: DPO alone on Public/Light (32.3k pairs) with β=0.1—outperforming SFT→DPO. Evaluated with LightEval accuracy and AlpacaEval (GPT-4 as judge). Reward model reached 64.01% on M1-Bench, 88.82% on Reward/Stem-Bench vs. 45.65% / 57.86% for base Qwen.
  • MCQA (Multiple-Choice QA): Trained three base variants (large 322k open-ended, medium with NuminaMath, small curated M1 1.3k). Applied LoRA adapters on 1k SciQ samples; best result: Quality M1 + LoRA (47.6% MMLU-STEM, 29.9% GPQA). LoRA on smaller, high-quality data beat full SFT on larger datasets—few adapter weights learn MCQ format without destabilizing the base model.
  • Quantized model: Post-training quantization via BitsAndBytes (8bit/4bit). LoRA16 on MMLU with prompt-in-loss: 4bit kept 44.7% accuracy vs. 49.7% bf16 while reducing VRAM. SmoothQuant and GPTQ (llmcompressor) underperformed. Quantization preserved accuracy for deployment on constrained devices.
  • RAG: Built 12 retrieval corpora (QA-formatted, Wikipedia STEM, OpenStax textbook excerpts, hybrid OpenStax+SciQ+Wiki). Dense retrieval with thenlper/gte-small, top-k=5. Surprisingly, RAG consistently hurt accuracy vs. no-retrieval baseline (40.14%)—retrieved passages added noise; QA-formatted led to answer copying. Fine-tuning with retrieved docs in prompt (RA-DIT style) did not improve performance. OpenStax Enhanced reached 33.81%, still below baseline.

Evaluation used LightEval on MMLU-STEM, GPQA, ARC, SciQ, and custom benchmarks. LLM-as-a-judge (AlpacaEval) validated preference alignment. Conclusion: sub-billion-parameter models can deliver strong STEM performance with the right strategy—DPO for alignment, LoRA for efficient format adaptation, quantization for deployment—but small models need different recipes than large ones.

ML for Early Detection of Growth Faltering

Applied ML · Healthcare · Large-Scale Data

Predicting malnutrition at a child's next visit from prior consultations—using 16M+ longitudinal records from Burkina Faso to enable early intervention in resource-limited settings.

Paper Code & data not publicly available (academic/confidentiality constraints)
Click for details

Course: CS-433 Machine Learning — in collaboration with the Machine Learning and Optimization Laboratory (MLO) at EPFL. Group project of 3.

Malnutrition contributes to over 45% of deaths in children under five; the burden is most severe in resource-limited regions. This project leverages machine learning to predict malnutrition status at a child's next medical visit using data from prior consultations—enabling early detection and targeted interventions.

We use a dataset from Terre des Hommes containing ~16 million records of medical consultations for children under five in Burkina Faso. Features include anthropometric data (weight, height, mid-upper arm circumference, Z-scores), demographics, clinical conditions, and feeding practices. Malnutrition states: severe (with/without complications), moderate, or none—with severe cases under 2%, leading to strong class imbalance.

  • Temporal framework: We construct a longitudinal setup where data from previous visits are inputs and the next visit's result is the target. We predict within the next four months to exclude extreme visit intervals. Two strategies were explored: fixed-interval analysis and time-aware modeling (time-to-next-visit as a feature).
  • Preprocessing: Feature selection; imputation (mode for categorical, mean for numerical); SMOTE for class balancing; one-hot encoding. We compare binary (malnutrition vs. no malnutrition) and multi-class setups—binary performs better due to reduced complexity.
  • Models: Logistic Regression (baseline), Random Forest, and LightGBM. Evaluated with accuracy and macro F1; F1 is critical to balance precision (avoid false alarms) and recall (catch at-risk children).
  • Visit frequency: Performance improves with more historical visits. Best configuration: 3 visits (2 prior + target), with models trained on non-improving sequences only (worsening or unchanged malnutrition)—focusing on the most vulnerable cases. F1 improves significantly on this subset.

Results: LightGBM outperformed Random Forest and Logistic Regression. On the full dataset, the best model (3 visits, binary target, non-improving sequences) achieved 96.68% accuracy, 76.54% F1, 68.88% recall, and 98.22% precision. Multi-class: 96.14% accuracy, 52.15% macro F1. Our main contribution is extending prior work (BDHS, Ethiopian studies) to longitudinal data with multi-visit prediction. Future work could explore LSTMs or Transformers for richer temporal modeling. Code and data are not publicly available due to confidentiality.

Secure Distributed Computing

Decentralized Systems Engineering · Distributed Systems · Cryptography · FHE

Train a global model across devices without anyone—including the server—ever seeing individual updates. Extends Peerster with xMK-CKKS homomorphic encryption for private aggregation.

Report Code not publicly available (academic reasons)
Click for details

Course: CS-438 Decentralized Systems Engineering — taught by Prof. Pierluca Borsò and Prof. Bryan Ford. Group project of 3.

Built on the Peerster framework—a decentralized P2P system in Go supporting messaging, file sharing, and consensus—this project extended it with privacy-preserving federated learning. The goal: train a global model across devices without any participant (including the aggregation server) ever seeing individual model updates. Traditional federated learning still leaks information via shared updates; we addressed this using xMK-CKKS (multi-key homomorphic encryption) so that only the aggregated sum can be decrypted.

The system operates under an honest-but-curious threat model. Each federated learning round follows three phases:

  • Phase 1 — Round initialization: The server proposes an aggregation round with ProposeAggregationRound (RoundParams); devices respond with AcceptAggregationRound and their public keys. The server waits until a target number of devices participate.
  • Phase 2 — Encrypted training: The server sends InitAggregationRound (aggregated public keys, global weights). Devices train locally, encrypt their updates, and send ModelUpdate (encrypted weights). Alive heartbeats detect failures; early termination is supported.
  • Phase 3 — Decryption and averaging: The server aggregates ciphertexts and broadcasts ModelAggregation. Devices send DecryptionShare; the server combines shares, recovers the sum, averages, and terminates with the new global weights.

Implementation details: message handlers for all aggregation messages; asynchronous training in Go routines; unicast for accept-messages to avoid congestion; broadcast for others; fault monitoring via AliveMessage. Encryption latency scales with model size and number of peers. Tested under varying topologies (normal, jammed, multiple concurrent rounds) with 80.7% code coverage. Best project grade in class (1st among 29 groups) in a cohort of 100+ Master's students.

Proximity — Decentralized Trust Graph

SUI x BSA Hackathon 2024 · $20K Prize Pool

Decentralized proximity detection—Bluetooth for presence verification, blockchain for real-time, immutable on-chain trust graphs of who was physically near whom.

GitHub
Click for details

Proximity is a decentralized trust graph for presence verification, built in 24 hours for the SUI x BSA Hackathon 2024. The system uses Bluetooth for proximity detection and blockchain to record verifiable, immutable presence proofs—creating a real-time on-chain trust graph of who was physically near whom.

The architecture spans three components: a Java backend (proximity-server) for coordination and Bluetooth logic, a web demo (proximity-web-demo) for user interaction, and Move smart contracts (proximity-move) on the Sui blockchain for storing and verifying P2P presence proofs. Achieved 2nd place among 30+ international teams.

3rd Place

TrustLink — Secure Smart Contract Escrow

SUI x BSA Hackathon 2023 · $25K Prize Pool

Trustless Web3 escrow—smart contracts on Sui/Move with modular object-ownership for multi-party exchanges with conditional release and expiration.

Code
Click for details

TrustLink is a secure escrow system built in 24 hours for the Sui Blockchain Hackathon 2023. It enables trusted exchanges between multiple parties with specific roles, requirements, and conditional actions—all enforced on-chain.

The Move smart contracts define modular structures: TrustedContract (immutable rules), TrustedContractHandler (mutable state), ContractDescriptor (steps, requirements, actions, expiration). Participants put items into the contract, confirm steps, and items are released only when all requirements are met. Supports expiration handling, lock/unlock of assets, and a confirmation key mechanism to bridge on-chain logic with real-world verification. 3rd place in the international competition.
